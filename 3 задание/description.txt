ЧТОБЫ ЗАПУСТИТЬ ОБУЧЕНИЕ МОДЕЛИ:
файлы train.py; model.py; NEREL/train должны находиться в одной директории
Запуск обучения командой: python3 train.py
После отработки этой команды модель будет автоматически сохранена в той же директории по пути models/model.pkl.gz


1. parser.py
Для начала напишем парсер для набора данных, он берет на вход название папки в которй и находится нужный нам набор данных, а на выходе выдает кортеж (text, entities), text - список тексов, entities - список списков сущностей для каждого из текстов
Суть метода в том, что он проходится по всем файлам с расширением .txt, вытаскивает из него сам текст, после чего просматривается файл с таким же именем, только с расширением .ann 
Файл .ann просматривается построчно, выбираются только строки начинающиеся с 'T' так как только они относятся к сущностям.
Каждая строка выглядит примерно так: T5	PERSON 65 76	Барак Обама
В них соотвтетственно PERSON - тип сущности
					  65 - начальная позиция
					  76 - коненая позиция
					  Барак Обама - сам фрагмент текста, относящийся к сущности
Сначала разделяем строку на 2 части по символу табуляции и берем левую часть, разделяем ее по пробелу и сохраняем (тип, начало, конец). Пропускаем разрывные сущности, перечисленные через ';'.


2. solution.py
метод __init__ загружает обученную модель из "models/model.pkl.gz"
метод predict вызывает метод модели predict_docs


3. model.py
Задача NER - это задача классификации. Для ее решения чаще всего используют логистическую регрессю, метод опорных векторов, случайный лес или бустинг.
В качетсве моего метода решения я выбрала использование логистической регрессии ищ-за ее простоты реализации.

метод tokenize_with_offsets: Разделяет текст на токены и превращает каждый в Token.

метод word_shape: Преобразовывает слова в абстрактные шаблоны, A...Z-> X, a...z-> x, цифра-> d, остальные символы как есть. 

метод token_features: создает набор признаков (слово как есть, в нижнем регистре, заглавное ли, является цифрой, длина, префиксы 1-3 символа, суффиксы 1-3 символа) для одного токена. И также добавляет соседние токены.

метод spans_from_iob: метод пробегает по всем токенам и их меткам, 
На B... - закрывает старую сущность (если была) и начинает новую, 
На I... - продолжает только если тип совпадает, 
На O или любой другой случай - закрывает сущность,
В конце — закрывает последнюю сущность, если она открыта.
То есть это обратная функция, которая превращает предсказанные моделью IOB-метки обратно в интервалы (start, end, type)


Класс NERModel:

метод __init__: инициализирует саму модель
FeatureHasher - превращает словесные признаки в числовой вектор, 
label_encoder - кодирует метки, 
clf - будет логистической регрессией.

метод featurize_docs:
для каждого документа берем текст, токенизируем, получаем список токенов, заводим для каждого токена метку O по умолчанию.
Запускаем цикл по аннотациям сущностей (start,end,type), помечаем первый токен в сущности как B-, остальные как I-.
Для каждого токена собирается словарь признаков для токена
Превращаем пары ключ-значение в строки "k=v", потому что FeatureHasher ожидает последовательность строк
Хэшируем все признаки в разреженную матрицу признаков.

метод fit: 
получаем признаки и метки для всех токенов в docs; строим сопоставление строки~числа и переводим все метки в числа; 
создаём модель логистической регрессии с параметрами:
solver='saga' - оптимизатор, рабочий с разреженными матрицами,
penalty='l2' - регуляризацияб
C - обратная регуляризация,
max_iter - максимальное число итераций оптимизатора,
n_jobs=1 - использовать один поток.
Далее обучаем модель на данных и сохраняем её внутри объекта.

метод predict_docs:
токенизируем каждый входной текст, далее для каждого токена собираются признаки, и хэшируются в матрицу X.
если модель не обучена возвращаем пустое множество.
Иначе делаем предсказание меток в виде чисел, переводим числа обратно в строковые метки, собираем интервалы (start,end,type) из меток, возвращаем список множеств сущностей для всех текстов.

метод save: сохраняет полученную модель

load: загружает и распаковывает модель


4. train.py
Функция build_docs_from_parser: пробегаемся по множеству пар текст+набор его сущностей. Для каждой сущности создаем словарь из: начало, конец, тип. То есть данный метод возвращает список словарей из пар текст + словарь сущностей для него.
В основной части: Загружаются train набор, инициализируется сама модель (размер хэша подбирался экспериментально, 4096 и 8192 - лучше по размеру и нормально по F1, но для 16384 есть риск с размерами итоговой модели, но результат немного повыше. window - размер контекстного окна, нужен для контекстной зависимости, по 2 слова слева и справа - оптимально) и запускается обучение на 300 итераций (недолго и приемлемый F1, при 400 и при 200 был немного меньше). Сразу после обучения модель сохраняется.


5. eval.py
Загружается обученная модель и тестовый набор данных и высчитывается средний F1 на нём
